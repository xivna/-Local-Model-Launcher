# Local LLM Manager (Windows) 🖥️

这是一个基于 Windows 批处理脚本编写的轻量级本地大模型管理器。它主要用于管理 `llama.cpp` (llama-server) 的启动、切换与端口控制，旨在简化本地部署多模型的流程。

## 特性 ✨
- **多模型支持**：如示例中同时配置了 Qwen3.5 与 Gemma-4 系列的不同量化版本（包含 Thinking 与非 Thinking 模式）。只需提前预设好启动指令。
- **智能端口管理**：内置端口检测机制，切换模型前自动清理旧进程，防止端口冲突。
- **可视化菜单**：通过文本界面选择模型组与具体文件，支持上下文菜单操作。
- **参数可配**：启动命令中的温度、采样等参数已硬编码展示，以供参考。
- 注：RTX4070 12G VRAM + 32G RAM跑Qwen35B-A3B可达34+ Tokens/s。

## 环境要求 📌
1. **操作系统**: Windows (XP 以上版本，推荐 Win10/Win11)。
2. **运行依赖**: 
   - 确保系统中已安装 `llama-server.exe`。
   - 当前脚本默认指向：`C:\LLama.cpp\llama-server.exe`，切记启动脚本前自行配置。
3. **模型文件**: 需要准备好 GGUF 格式的模型文件。

## 快速开始 🚀

### 1. 下载与解压
将项目文件`llm_manager.bat`下载到本地即可。

### 2. 配置路径 ⚠️ 重要
**脚本中默认的路径可能不适用于你的环境**。请务必在编辑脚本时修改以下部分：
- **LLM Server 位置**: `set MODEL_CMD_!IDX!="C:\LLama.cpp\llama-server.exe"` (搜索并替换为你的实际路径)
- **模型文件路径**: 如 `E:\Qwen3.5...\Qwen3.5-0.8B-UD-Q6_K_XL.gguf`，请根据你的实际存放位置修改。

> 💡 **建议**：为了方便维护，建议在首次运行时将脚本保存一份备份，然后使用记事本或编辑器搜索 `.exe` 和 `.gguf` 关键词进行全局替换。

### 3. 运行
直接双击 `llm_manager.bat` 或在命令行中运行：
```bash
.\llm_manager.bat
```

## 使用说明 📖
程序启动后，主界面提供以下操作：
1. **系列选择**: 选择想运行的模型组（如 Qwen3.5, gemma-4-E4B）。
2. **模型选择**: 在组内选择具体版本（包含 Non-thinking / Thinking 模式差异）。
3. **运行中状态**: 如果已有模型在监听 `11436` 端口，系统会自动停止旧服务再启动新的。

### 菜单逻辑
- `[0]`：退出当前程序（会强制关闭后台运行的 LLM）。
- `[2]`：仅切换回菜单但不退出管理器。
- 日志窗口会在新终端打开显示 `llama-server` 输出，方便查看推理信息。

## 自定义模型 🛠️
如果你需要添加新的模型，只需在脚本顶部的 **“模型配置区”** 进行增删：
```batch
:: ── 新增模型组示例 ────────────────────────────────────────────── 
set /a GIDX+=1 & set GRP_NAME_!GIDX!=你的新模型名
:: ── 添加具体模型参数 ────────────────────────────────────────────
set /a IDX+=1 & set MODEL_GRP_!IDX!=!GIDX!
set MODEL_NAME_!IDX!= [ 描述 ] 模型名称 -- 说明 
set MODEL_CMD_!IDX!="C:\LLama.cpp\llama-server.exe" -m "你的.gguf 路径" ...参数...
```
添加后保存脚本并运行即可生效。

## 注意事项 🛑
- **资源占用**: 本脚本未包含显式内存限制，启动大型模型（如 31B）前请确保硬件资源充足。
- **安全**: 脚本以管理员权限或普通用户权限均可运行，但涉及 `taskkill` 操作，建议在无其他关键服务占用端口环境下使用。
- **版本兼容性**: 
  - 注意 `-fit`, `-n-cpu-moe` 等参数依赖于你编译的 `llama-server` 版本。如果参数报错，请查阅 llama.cpp 文档确认当前 server 是否支持该指令。
